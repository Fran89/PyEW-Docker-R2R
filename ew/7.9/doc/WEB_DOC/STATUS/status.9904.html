Earthworm Version 3.91 has been released.

You can download the release from our new ftp site:
ghtftp.cr.usgs.gov/pub/OUTGOING/earthworm/release_v3.91

NEW MODULES

q2ew: module to convert data acquired by the Quanterra data loggers into earthworm packets. The Quanterra data is imported into a Solaris machine via the comserv program. The module q2ew interfaces to comserv and converts the miniSEED packets to Earthworm trace_buf packets. Written by Paul Friberg & Sid Hellman of ISTI. ew2seisvole: Listens for hypo_sum messages, and produces files containing cubic messages in a specified directory. Seisvole can be configured to retrieve the cubic messages from the specified directory and display on it's maps. A readme file can be found in the ew2seisvole source directory.

IMPORTANT BUG FIXES

carlstatrig: Fixed bug in addtrace.c where gapSize was calculated incorrectly. This was causing the STA/LTA values to become NaNs (Not A Number) which prevented triggers from that channel. There have been many more bug fixes and enhancements to Earthworm modules with v3.91. See the release notes which accompany the release for details.

IMPORTANT NOTICE TO ADSEND USERS

Earthworm module 'adsend' can freeze an NT system if restarted using the Earthworm RESTARTME mechanism. We recommend you do not use this feature with 'adsend'.

CONTRIBUTED SOFTWARE

Many Earthworm modules have been developed at regional networks to meet their specific needs. Since some of these may be useful to other networks, we make these available at the Earthworm ftp site in the Contributed_software directory. Currently, you will find modules developed at USGS Menlo Park, University of Memphis, University of Washington and University of Alaska, Fairbanks.
